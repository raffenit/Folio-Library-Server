# Clean Code & SOLID Standards
All code generated must adhere to these industry standards:
- **S.O.L.I.D**: Especially the Single Responsibility Principle (functions should do one thing).
- **Clean Code (Robert C. Martin)**: 
  - Use intention-revealing names.
  - Functions should be small (prefer < 20 lines).
  - No side effects in pure functions.
- **DRY (Don't Repeat Yourself)**: If you see pattern repetition, propose a shared utility or abstract component.
