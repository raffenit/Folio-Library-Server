# The Development Protocol
Whenever a user asks for a code change:
1. **Think Phase**: Use `<thinking>` tags to draft the logic.
2. **Standard Check**: Validate the draft against the "React & TS Clean Code Standards" and "Clean Code Standards" rules.
3. **Internal Audit**: Run the "React & TS Logic Auditor" logic on your own draft.
4. **Correction**: If a flaw is found, start over.
5. **Output**: Only then, present the final, audited code to the user.
